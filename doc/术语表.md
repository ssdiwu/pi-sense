# pi-sense 术语表

`pi-sense` 是 Pi coding agent 的媒体 handoff 扩展：当 active model（当前模型）不支持图像输入时，自动用用户指定的模型描述媒体内容；本地视频也复用同一 non-vision handoff 目标判定，再把文字描述注入上下文，让纯文本模型也能"看见"媒体。

## Language

**handoff（代理/转手）**：
当 active model 无法直接处理某种输入媒体时，扩展把媒体交给另一个能力模型处理，再把结果返回给 active model 的机制。
_Avoid_: proxy、fallback

**active model（当前模型）**：
Pi 会话中当前被选中的 LLM，即接收扩展注入描述文本的模型。
_Avoid_: main model、target model

**vision model（视觉模型）**：
支持图像输入的多模态模型，在 `pi-sense` 中用于描述图像帧或视频帧。
_Avoid_: image model、multimodal model（后者范围过大）

**video model（视频模型）**：
`pi-sense` 中专门用于描述视频帧序列的模型。默认复用 `vision model`，但可单独配置。
_Avoid_: video understanding model

**原生视频理解（native video understanding）**：
模型本身直接接收视频文件输入并理解内容的能力（如 MiniMax-M3、Gemini、Grok），通过 provider 特定的 `video_url`/`inline_data` 块传入。`pi-sense` 的原生路线用它做内容理解（what），但模型的时间感知不可靠。
_Avoid_: video input、video QA

**时间感知（temporal awareness）**：
对视频中事件发生时刻的精确对应（“在 1:30 发生了什么”）。原生模型的时间感知不可靠（时间戳与真实时长系统性偏差），因此 `pi-sense` 在需要时间感知时走抽帧+ASR 路线——插件自己控制帧采样时间点和 ASR segment 时间戳，建立确定性的时间-内容对应。
_Avoid_: 时间戳、时间线、timeline（这些是产物，不是能力）

**ASR（自动语音识别）**：
把视频/音频中的语音转换成带时间戳文本的过程。`pi-sense` 默认优先调用外部 `whisper-cli`，再 fallback 到本地 `faster-whisper` 虚拟环境。
_Avoid_: 转录器、语音识别器

**scene-cut（场景切换检测）**：
基于 ffmpeg 的画面变化检测，抽取出视频中出现显著视觉跳变的帧。
_Avoid_: keyframe、scene detection

**adaptive sampling（自适应采样）**：
在全局采样基础上，检测相邻帧描述的语义跳变或时间间隔过大，对局部区间加密补帧的机制。第一版预留开关，默认关闭。
_Avoid_: 智能抽帧、动态采样

**media block（媒体块）**：
Pi-ai 消息内容中的非文本内容单元，如 `{ type: "image", data, mimeType }` 或未来可能支持的 `{ type: "video" }`。
_Avoid_: 附件、文件块

**description fence（描述围栏）**：
扩展注入上下文时使用的结构化文本边界，如 `[Image: ...]` 与 `[Video: ...]`，用于区分模型生成的描述文本与原始媒体。
_Avoid_: 描述标签、描述框

**SenseConfig（sense 配置）**：
`pi-sense` 的持久化配置文件 `~/.pi/agent/pi-sense.json` 对应的数据结构。
_Avoid_: dvision config、vision config
